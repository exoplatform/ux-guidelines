Upload Pattern

Basic style and behavior for uploading file function.

  • Overview
  • What Problem Does This Resolve?
  • When to Use This Pattern?
  • What is the Solution?
  • Why Use This Pattern?

Overview

Upload function allows to attaching files to specific content or changing the avatar of a page or personal image.

What Problem Does This Solve?

The user needs may want attach related material to their content such as event, topic, post, wiki... or upload files for storage or sharing. The upload function is a tool to do this.

When to Use This Pattern?

With any systems that need to upload files from local for changing avatar, file storge, file sharing, importing data, upload function must be used.

Status

Status Description
No file is uploaded by default The Upload popup is displayed with default style.
A file is being uploaded There is a file name is displayed with progress bar that show the percentage of uploading.There is cancel button to stop uploading process
A file is uploaded completely The progress bar is disappeared. There is a trash icon to remove the uploaded file.

Rules

UP-01 Upload button or symbol is easy to see.
UP-02 Only one file can be selected at once.
UP-03 Multiple files are being uploaded.
UP-04 The uploading percentage number must be matched with the progress bar status.
UP-05 The uploaded files are saved into database only when the Save button are saved.
UP-06 The file title must be cut off with 20 character and "..." in case having long title.

Why to Use This Pattern?

The pattern is to make sure the consistency among applications in the way to uploading files.
