Headings

Typography is about the typefaces such as font style, font size, color and the space around and through them to create a whole design.
Good typography makes good design for a application. Each typeface has specific purpose in use that help to recognize the role or the importance of a specific content.
Each application has general typography commonly in use to show its own style and also obtain the consistency and professionalism. Therefore, the typography should be applied to overall application.
All HTML headings, <h1> through <h5> are available. Besides, the cover title or page title can be defined by large font size

The header tags are important to SEO because the main content of a heading are considered as key term for search engine. There are some notices:
Use and apply correct header tags.
Never jump from h1 to h3, h2 to h4.
No need to use all h1...h5 in a page.
Use only 1 Header cover.
Use only 1 h1 in case there is no heading cover with bigger size.
Concise content for heading title to help decide the keyterm easily.

Emphasis

Make use of HTML's default emphasis tags with lightweight styles.

<small>

For de-emphasizing inline or blocks of text, or explaination for any word at the end of a page, use the small tag.

This line of text is meant to be treated as fine print.

Bold

For emphasizing a snippet of text with important

Italics

For emphasizing a snippet of text with stress

Heads up! Feel free to use <b> and <i> in HTML5. <b> is meant to highlight words or phrases without conveying additional importance while <i> is mostly for voice, technical terms, etc.

Blockquotes

For quoting blocks of content from another source within your document.

Default blockquote

Wrap <blockquote> around any HTML as the quote. For straight quotes we recommend a <p>.

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer posuere erat a ante.

Blockquote options

Style and content changes for simple variations on a standard blockquote.

Naming a source

Add <small> tag for identifying the source. Wrap the name of the source work in <cite>.

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer posuere erat a ante.

Inline

Basic block

Use <pre> for multiple lines of code. Be sure to escape any angle brackets in the code for proper rendering.

<p>Sample text here...</p>

Heads up! Be sure to keep code within <pre> tags as close to the left as possible; it will render all tabs.

You may optionally add the .pre-scrollable class which will set a max-height of 350px and provide a y-axis scrollbar.
