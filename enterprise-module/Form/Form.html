Form

Basic CSS for input elements in a form.

  • Special fields
  • Basic input elements

Special fields

Based on Bootstrap, there are some special fields with the eXo Platform style.

Search field

Add the <input> inside the class uiSearchInput. 

<div class="uiSearchInput">
  <input id="inputValue" type="text" placeholder="Search">
</div>

This search type is a good choice for the simple search function.

Text field with button

Add button and input inside the class input-append.

<div class="input-append">
  <input type="text"><button class="btn" type="button">Action</button>
</div>

The inline form is suitable with simple function that is expected to display in one line to save spaces, such as login and newsletter.

Big Title Text Field

This text field has big size that can be used for special purposes, for example sign-in title, wiki title.

<div class="uiLargeTextField">
  <div class="input-xxxlarge">
    <input type="text" placeholder="Untitled">
  </div>
</div>

Basic input elements

Text field

Basic inputs are text fields that are supported with types: text, password, datetime, datetime-local, date, month, time, week, number, email, url, search, tel, and color.

The type is required at all times.

<input type="text" placeholder="Text input">

Input size

<div class="controls docs-input-sizes">
  <input class="input-mini" type="text" placeholder=".input-mini">
  <input class="input-small" type="text" placeholder=".input-small">
  <input class="input-medium" type="text" placeholder=".input-medium">
  <input class="input-large" type="text" placeholder=".input-large">
  <input class="input-xlarge" type="text" placeholder=".input-xlarge">
  <input class="input-xxlarge" type="text" placeholder=".input-xxlarge">
</div>

Validation states

<div class="control-group error">
  <label class="control-label" for="inputError">Input with error</label>
  <div class="controls">
    <input type="text" id="inputError">
  </div>
</div>
<div class="control-group success">
  <label class="control-label" for="inputSuccess">Input with success</label>
  <div class="controls">
    <input type="text" id="inputSuccess">
  </div>
</div>

Disabled inputs

Add the disabled attribute to disable an input field.

<input class="input-xlarge" id="disabledInput" type="text" placeholder="Disabled input here…" disabled>

Textarea

A textarea allows more space for data input. Change the rows attribute as necessary.

<textarea rows="3"></textarea>

Checkboxes and radios

Switch Radio

By using the iphone-style-checkboxes.js provided by eXo Platform Team, there is a nice switch radio:

<div class="uiSwitchBtn" style="width: 60px;">
  <div class="iPhoneCheckContainer">
    <input type="checkbox" checked="checked" class="providerAction yesno enable" data-no="NO" data-yes="YES" value="true">
    <label class="switchBtnLabelOff">
      <span>NO</span>
    </label>
    <label class="switchBtnLabelOn">
      <span>YES</span>
    </label>
    <div class="switchBtnHandle">
      <div class="switchBtnHandleRight">
        <div class="switchBtnHandleCenter"></div>
      </div>
    </div>
  </div>
</div>

Switch button without label using only css:

<label class="uiSwitcher">
  <input type="checkbox" checked="checked" />
  <span class="switcher-slider"></span>
</label>

Selects

The default or multiple="multiple" options can be selected at once.


<select>
  <option>1</option>
  <option>2</option>
  <option>3</option>
  <option>4</option>
  <option>5</option>
</select>

<select multiple="multiple">
  <option>1</option>
  <option>2</option>
  <option>3</option>
  <option>4</option>
  <option>5</option>
</select>
