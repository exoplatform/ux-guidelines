Breadcrumb

A navigation aid to keep track of location in an application or a website.

  • Overview
  • Solution

Overview

A breadcrumb is a navigation or a path link in the user interface. It allows users to keep track of their position in an application, document or website. Also, breadcrumbs allows users to navigate to everywhere only in a click.

When and why to use?

Breadcrumbs should be used when the structure of the site is divided into subsections.
Breadcrumbs provide users with information where they are in a site/application, and shortcut links to jump to previously viewed pages, allow users to know the structure of a site/application. So, it should be used in all sub-pages.

Status

Status Description
Standard Status The label of a section in a hierarchical path that users can click to go to the home or previous/parent page of the being viewed page.
Being Viewed Status The label of the current page is at the end of breadcrumb and is not linked.

Rules

BR-01 No breadcrumbs at the homepage.
BR-02 Breadcrumbs are on the top of a page content.
BR-03 Breadcrumb label is the page content title.
BR-04 Breadcrumbs label length is limited to 20 characters.
BR-05 Hyperlink all labels except the last one and breadcrumbs icon.
BR-06 Hyperlink should be styled the same regardless of whether it has been visited or not.
BR-07 Breadcrumb box must cover the breadcrumb path.
BR-08 Breadcrumb length must be auto-resized and has the same minimum limitation in comparison to its size.
BR-09 Breadcrumb is broken in case it is longer than the minimum limitation.
BR-10 There are 2 spaces before and after each breadcrumb divider.
