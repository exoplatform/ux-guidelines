Message

How to dialog with users and how to display different message kinds to them. What and when should they be used?

  • Overview
  • Error message
  • Warning message
  • Information message
  • Confirmation message

Overview

Message is a short communication transmitted by words and signals. In applications, messages are considered as "feedbacks" responding to end-users, so they play an important role in the development process as well success of applications. To avoid unclear and incorrect messages, you need to clarify usage of message types and how to display them.

In eXo Platform, messages are grouped into 4 types:

Error
Warning
Information
Confirmation

In eXo Platform, message(s) can be displayed in a single (only one message) or multiple (more than 1 message) way.

Error message

An error message alerts users of a problem that has already occurred. Effective error messages should be relevant, brief, clear, specific, courteous about the followings:

Problem - States that a problem occurred. For example: Your connection is time out.
Cause - Explains why the problem occurred. For example: Your chosen Username already exists.
Solution - Provides suggestions for users to fix the problem. For example: Please input a valid email format, such as yourname@example.com.

  • Your email address is incorrect. Please input a valid one!

Warning message

A warning message is to alert users of a condition that might cause a problem in the future. A good warning message should:

Involve risk - Alerts users of something significant. For example: Are you sure to delete the driver and all its data?
Have immediate relevance - Gives clear reasons of warning. For example: Your account will be blocked after 5 times of incorrect password to prevent hackers!
Be clear and specific - Defines sources of warning, potential occurrences, and what users should do. For example: Your password is not strong enough. It should have both texts and numbers.

  • Your account will be blocked after 5 times of incorrect password to prevent hackers!

Information message

An information message is to alert users of useful and relevant, but not critical information. It is considered as a notification that lets users know specific information. For example: Your account has been registered successfully. Please go to your email address to activate your account.
A good information message should be:
Useful - Lets users know something new or has been done successfully.
Optional - Shows suggestions (such as version upgrade or new functions) that are not mandatory.
Relevant - Shows helpful information that users might not have known.

Confirmation message

A confirmation message is to ask if users want to proceed with an action. The confirmation can be displayed in 2 ways: default browser pop-up or customized pop-up window like the template below.
For important changes (for example, deleting a forum that causes all its topics and posts to be lost), the confirmation should be a warning that is displayed in a customized pop-up rather than browser pop-up.
A good confirmation message should give the followings:
Obvious result of an action initiated by the user.
Relevant verification if the user wants to proceed with the action.
Easy-to-understand question with two or more responses (for example, OK - Cancel, Yes - No).

Multiple messages

eXo Platform allows displaying multiple messages at the same time following the order priority: Error - Warning - Information.

Template

Content of the first displayed message type is expanded by default. Each message type can be clicked to show/hide flexibly. The label of each message type is in the format: {Number} Error(s)/Warning(s)/Information. For example: 5 Errors, 2 Warnings, or 1 Information.

Alert
  • 3 Errors
    • - The field "First Name" is required.
    • - The field "Last Name" is required.
    • - Your email address is invalid. Please enter a different address.

    1 Warning
    • - Your password is not strong enough.

    1 Information
    • - Your registration is completed. Please check your email inbox to activate your account!
OK
