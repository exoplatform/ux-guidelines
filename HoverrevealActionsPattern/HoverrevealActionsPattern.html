Hover reveal Actions Pattern

Hiding tools by default and revealing them on demand to make content cleaner

  • Description
  • Overview
  • What Problem Does This Solve?
  • When to Use This Pattern?
  • What's the Solution?
  • Object with few actions
  • Object with multiple actions
  • Why Use This Pattern?

Overview

Hover-Reveal Tools allow users to interact directly with the content. Instead of making Contextual Tools always visible, we can show them on demand by mouse over an object.
Hiding tools by default and revealing them on demand decreases the visual noise in the user interface and makes content cleaner and less cluttered.
When using Hover-Reveal Tools, discoverability might be an issue. As tools are shown only when hovering over elements, it might be difficult to discover this functionality.

What Problem Does This Solve?

Hover reveal tool makes user interface cleaner and provide quick and smart way to do actions on site contents.

When to Use This Pattern?

  • Use when doing actions quickly on objects
  • Use when there are similar objects that having same actions.
  • Use when desiring to have simple user interface.

    • What's the Solution?

    Hover reveal action tool is displayed when mouse over a specific object. Depending on the characteristic of the object, the effects may be different. Group by the number of actions, we can have 2 main kind of objects: object with few actions and object with multiple actions

    Object with few actions

    With the object having few actions, the action link/button/icon can be displayed directly when mouse over The samples below are suitable in case the number of actions less than 4.
